# 项目上下文

## 项目目标
本项目用于在一台中心服务器上统一纳管多台内网服务器上的服务，并通过 Web 页面查看状态、失败原因和运维开关。

当前目标场景是：
- 程序部署在服务器 A。
- 由 A 对 A、B、C、D 等服务器上的服务做心跳检测。
- 在本地 Win7 机器的浏览器中查看页面。

## 当前实现
- 检测方式
  - HTTP/HTTPS 健康检查。
  - 文件上传类业务检测。
- 运维方式
  - 远端 SSH 执行命令。
  - 本机 `localproc` 子进程启停。
- Web 能力
  - 登录、用户管理、服务绑定。
  - 自动检测开关与频率配置。
  - 自动重启开关。
  - 禁用/启用服务。
  - 启动、停止、重启、立即检测。

## 主要目录
- `main.py`
  - 启动入口。
- `monitor/webapp.py`
  - Web 路由与管理员接口。
- `templates/`
  - 前端页面模板。
- `core/`
  - 调度、状态存储、日志、权限、运行时状态收敛。
- `services/`
  - 通用服务和插件服务实现。
- `config/services/`
  - 正式服务配置。
- `config/samples/`
  - 样例配置。
- `archive/dev_tools/`
  - 开发和回归脚本。

## 当前约定
- 新服务建议先 `auto_check: false`，由管理员确认后再开启自动检测。
- “只监控/可维护”和“自动重启”是两套不同开关：
  - `ops_enabled` 决定是否允许启停/重启以及自动重启是否可执行。
  - `on_failure` 决定失败时是仅提示还是尝试重启。
- 页面显示与调度器行为必须共用同一套运行时状态收敛逻辑，入口在 `core/runtime_state.py`。
